# 🚀 دليل نشر نظام مكتب الرفاه (النسخة الإبداعية) على GitHub Pages

هذا الدليل سيساعدك على نشر **النسخة الإبداعية الكاملة** من نظام مكتب الرفاه كموقع ويب دائم ومجاني على **GitHub Pages**.

---

## ✨ ما الجديد في النسخة الإبداعية (V3)؟

هذه النسخة تتضمن ميزات احترافية متقدمة:
- 💱 **محول عملات ذكي** مع أسعار صرف قابلة للتحديث
- 📋 **سجل النشاطات الكامل** لتتبع جميع العمليات
- 🎨 **تأثيرات حركية فائقة الجودة** وواجهة مستخدم عصرية
- 📊 **إحصائيات ذكية** وتحليلات متقدمة
- 🔐 **نظام أمان محسّن** مع إدارة مستخدمين متكاملة

---

## الخطوة 1️⃣: إنشاء حساب GitHub (إذا لم تكن تملك واحداً)

1. اذهب إلى [GitHub.com](https://github.com)
2. انقر على **"Sign up"** (التسجيل)
3. أدخل بريدك الإلكتروني واسم المستخدم وكلمة المرور
4. أكمل التحقق من البريد الإلكتروني
5. تم! أنت الآن لديك حساب GitHub

---

## الخطوة 2️⃣: إنشاء مستودع جديد (Repository)

### الطريقة السهلة (عبر الموقع):

1. بعد تسجيل الدخول، انقر على **"+"** في الزاوية العلوية اليسرى
2. اختر **"New repository"**
3. ملأ البيانات التالية:
   - **Repository name**: `alrefah-accounting` (أو أي اسم تفضله)
   - **Description**: "نظام محاسبي متكامل - النسخة الإبداعية (Creative Edition)"
   - **Visibility**: اختر **"Public"** (عام) حتى يكون متاحاً للجميع
   - **Initialize this repository with**: اترك الخيارات الافتراضية
4. انقر **"Create repository"**

✅ تم إنشاء المستودع!

---

## الخطوة 3️⃣: رفع الملفات

### الطريقة 1: عبر الموقع (الأسهل للمبتدئين)

1. في صفحة المستودع الجديد، انقر على **"Add file"** → **"Upload files"**
2. اسحب وأفلت الملفات التالية:
   - `index.html`
   - `style.css`
   - `script.js`
   - `README.md`
   - `LICENSE`
   - `.gitignore`
3. في أسفل الصفحة، أضف رسالة في **"Commit message"**:
   ```
   Initial commit: Al-Refah Creative Edition V3
   ```
4. انقر **"Commit changes"**

✅ تم رفع الملفات!

### الطريقة 2: عبر سطر الأوامر (للمتقدمين)

إذا كان لديك Git مثبتاً على جهازك:

```bash
# استنساخ المستودع
git clone https://github.com/YOUR_USERNAME/alrefah-accounting.git
cd alrefah-accounting

# نسخ الملفات إلى المجلد
cp /path/to/index.html .
cp /path/to/style.css .
cp /path/to/script.js .
cp /path/to/README.md .
cp /path/to/LICENSE .
cp /path/to/.gitignore .

# إضافة الملفات
git add .

# حفظ التغييرات
git commit -m "Initial commit: Al-Refah Creative Edition V3"

# رفع إلى GitHub
git push origin main
```

---

## الخطوة 4️⃣: تفعيل GitHub Pages

هذه هي الخطوة الأهم لجعل الموقع متاحاً للجميع!

1. في صفحة المستودع، انقر على **"Settings"** (الإعدادات)
2. من القائمة الجانبية على اليسار، اختر **"Pages"**
3. تحت **"Source"**، اختر:
   - **Branch**: `main`
   - **Folder**: `/ (root)`
4. انقر **"Save"**

⏳ **انتظر 2-3 دقائق** حتى يتم نشر الموقع

---

## الخطوة 5️⃣: الحصول على الرابط الدائم

بعد انتظار 2-3 دقائق:

1. عد إلى صفحة **Settings** → **Pages**
2. ستجد رسالة خضراء تقول:
   ```
   Your site is live at: https://YOUR_USERNAME.github.io/alrefah-accounting
   ```

🎉 **هذا هو رابطك الدائم!**

---

## 🔄 تحديث الموقع

إذا أردت تعديل أي شيء في الموقع:

### عبر الموقع:
1. اذهب إلى المستودع
2. انقر على الملف الذي تريد تعديله
3. انقر على أيقونة **"Edit"** (قلم)
4. عدّل الملف
5. انقر **"Commit changes"**

### عبر سطر الأوامر:
```bash
cd alrefah-accounting

# عدّل الملفات كما تريد

git add .
git commit -m "تحديث: وصف التغييرات"
git push origin main
```

التحديثات ستظهر على الموقع خلال دقائق!

## 🎯 استخدام النسخة الإبداعية

### بيانات الدخول الافتراضية:
- **اسم المستخدم**: `admin`
- **كلمة المرور**: `123456`

### الميزات الرئيسية:

#### 1. لوحة التحكم (Dashboard)
- عرض ملخص الأرباح والمصروفات
- رسم بياني يومي للأرباح (آخر 30 يوم)
- أكثر البائعين ربحية

#### 2. الأرباح والمصروفات (Profits)
- تسجيل العمليات المالية
- تصفية حسب العملة والتاريخ
- حساب صافي الربح تلقائياً

#### 3. العملاء (Clients)
- إدارة قائمة العملاء
- تتبع الأرصدة لكل عميل
- إرسال كشف حساب عبر الواتساب مباشرة

#### 4. التقارير المحاسبية (Reports)
- ملخص شامل للميزانية
- توزيع الأرباح حسب العملة
- تصدير إلى Excel و PDF

#### 5. أدوات متقدمة (Advanced Tools) ⭐ **جديد**
- **محول العملات**: تحويل المبالغ بين الدولار والليرات
- **أسعار الصرف**: تحديث أسعار الصرف الحالية
- **إحصائيات ذكية**: تحليل الديون والعملاء

#### 6. سجل النشاطات (Audit Log) ⭐ **جديد**
- تتبع كل العمليات (دخول، خروج، إضافة، تعديل، حذف)
- معرفة من قام بأي عملية ومتى
- **متاح فقط للمدير**

#### 7. الإعدادات (Settings)
- إدارة المستخدمين (إنشاء، تعديل، حظر، حذف)
- تصدير واستيراد البيانات
- حذف جميع البيانات

## ⚠️ ملاحظات مهمة

1. **البيانات محفوظة محلياً**: جميع البيانات تُحفظ في متصفحك فقط، لا تُرسل إلى خادم خارجي.
2. **النسخ الاحتياطية**: استخدم ميزة "تصدير البيانات" من الإعدادات بانتظام.
3. **الخصوصية**: الموقع عام، لكن البيانات تبقى في جهازك.
4. **الأداء**: الموقع سيكون سريعاً جداً لأنه ملفات ثابتة.
5. **الأمان**: واجهة الدخول مؤمنة، وسجل النشاطات متاح فقط للمدير.

## 🆘 استكشاف الأخطاء

### الموقع لا يظهر؟
- تأكد من أن الملفات مرفوعة بشكل صحيح
- انتظر 5 دقائق إضافية
- تحديث الصفحة (Ctrl+F5 أو Cmd+Shift+R)

### الملفات لا تُحمّل؟
- تأكد من أن `index.html` موجود في المجلد الرئيسي
- تحقق من أسماء الملفات (يجب أن تكون صحيحة تماماً)

### الأسلوب لا يظهر بشكل صحيح؟
- امسح ذاكرة التخزين المؤقت (Cache)
- استخدم Ctrl+Shift+Delete لحذف الملفات المؤقتة

### الميزات لا تعمل؟
- تأكد من أن JavaScript مفعّل في متصفحك
- جرّب متصفح آخر (Chrome, Firefox, Safari)
